# PowerCo 모델링 실험 및 최종 선택 정리

## 1. 실험 목적

초기 Feature만으로 끝내지 않고 다음 질문을 순서대로 확인했다.

```text
기본 Feature만으로 충분한가?
        ↓
어떤 정보를 추가하면 이탈 고객 순위화가 좋아지는가?
        ↓
불균형 처리를 적용하는 것이 실제로 도움이 되는가?
        ↓
어떤 모델이 가장 안정적으로 고위험 고객을 선별하는가?
```

핵심 목표는 **모든 고객을 이탈/유지로 완벽하게 분류하는 것**이 아니라, 제한된 유지관리 자원으로 **이탈 위험이 높은 고객을 우선 선별하는 것**이다.

주요 평가 기준:
- PR-AUC: 불균형 데이터에서 위험 순위화 성능
- Precision / Recall / F1: 분류 성능
- Top 10% Recall: 상위 10%가 전체 이탈자의 몇 %를 포함하는지
- Top 10% Lift: 상위 10%의 이탈 고객 밀도가 무작위보다 몇 배 높은지

---

## 2. 기본 Feature A0

A0:
- 모델 Feature: 25개
- 계약·소비·가격·마진 중심

무가중치 LightGBM 기준:

| 지표 | A0 |
|---|---:|
| OOF PR-AUC | 0.275470 |
| ROC-AUC | 0.688853 |
| Top 10% Recall | 0.293392 |
| Top 10% Lift | 2.932415 |

무작위 PR-AUC 기준선이 약 0.097이므로 유효한 신호는 있었지만, 고위험 고객을 더 정교하게 선별할 여지가 있었다.

## 3. Feature Engineering 실험

### A1 — 누수 민감도 확인

확인 대상:
- `contract_end_within_3m`
- `forecast_*`
- 계약 관련 날짜 정보

목적:
- 특정 변수에 성능이 과도하게 의존하는지 확인
- 예측 시점에 사용할 수 있는 정보인지 점검

결론:
- 최종 Feature 세트로 선택하지 않음
- 날짜·예측값 Feature는 실제 운영 전 생성 시점 확인 필요

---

### A2 — 가격 시계열 확장

월별 가격에서 평균, 변동성, 추세, 최근 변화 등 다수의 통계 Feature를 추가했다.

결론:
- Feature 수는 크게 늘었지만 A3보다 PR-AUC가 높지 않았음
- 가격 정보 일부가 기존 변화율 Feature에 이미 포함되어 있었을 가능성
- 추가 Feature 대비 성능 개선 근거가 부족해 제외

---

### A3 — 계약 생애주기 정보 추가

A0에 계약 날짜 기반 Feature 12개를 추가했다.

주요 정보:
- 계약 유지 기간
- 계약 종료일까지 남은 기간
- 갱신일까지 남은 기간
- 상품 변경 후 경과 기간
- 계약 생애주기 진행 비율

최종 Feature:
- A0 25개
- 계약 날짜 Feature 12개
- 총 **37개**

동일한 무가중치 LightGBM 비교:

| 지표 | A0 | A3 | 변화 |
|---|---:|---:|---:|
| OOF PR-AUC | 0.275470 | **0.317581** | +0.042111 |
| ROC-AUC | 0.688853 | **0.713394** | +0.024541 |
| Top 10% Recall | 0.293392 | **0.333921** | +0.040529 |
| Top 10% Lift | 2.932415 | **3.337493** | +0.405078 |

- PR-AUC 상대 개선: 약 **15.3%**
- Top 10% Lift 상대 개선: 약 **13.8%**

### 핵심 인사이트

Feature를 많이 추가하는 것보다 **문제와 직접 연결된 계약 생애주기 정보**를 추가하는 것이 더 효과적이었다.

---

### A4 — 비율·노출도 Feature

소비량 대비 마진, 계약전력 대비 소비량 등 비율 Feature를 추가했다.

결론:
- A3보다 높은 PR-AUC를 만들지 못함
- 분모 0에 따른 구조적 결측과 극단적인 분포가 발생할 수 있음
- 원본 소비량·마진 Feature가 이미 주요 정보를 포함하고 있을 가능성

따라서 최종 데이터에는 포함하지 않았다.

---

## 4. Feature 선택 결론

```text
A0 기본 Feature
    ↓
A1 누수 민감도
A2 가격 시계열
A3 계약 생애주기
A4 비율·노출도
    ↓
A3 선택
```

최종 판단:
1. 계약 생애주기 정보가 가장 일관된 성능 개선을 보였다.
2. 57개 가격 Feature를 추가한 A2보다 12개 계약 Feature를 추가한 A3가 더 간결했다.
3. Feature 수를 늘리는 것보다 의미 있는 정보를 추가하는 것이 중요했다.

## 5. 클래스 불균형 처리 실험

이탈률이 약 9.7%이므로 다음 방법을 비교했다.

- 가중치 없음
- Class Weight
- Random Oversampling
- Balanced Random Forest

주요 결과:

| 방법 | OOF PR-AUC | Recall | Top 10% Lift |
|---|---:|---:|---:|
| **LightGBM 무가중치** | **0.317581** | 0.645815 | **3.337493** |
| LightGBM Class Weight | 0.291173 | 0.649339 | 3.152566 |
| LightGBM Random Oversampling | 0.285027 | **0.700441** | 3.126148 |
| RF Random Oversampling | 0.278713 | 0.744493 | 3.020475 |
| RF Class Weight | 0.278254 | 0.693392 | 2.994057 |
| RF 무가중치 | 0.268527 | 0.650220 | 2.932415 |

### 핵심 인사이트

Oversampling은 Recall을 높였지만:
- PR-AUC 하락
- Precision 하락
- Top-K 성능 하락

즉 더 많은 이탈자를 잡는 대신 유지 고객도 많이 위험군으로 포함했다.

최종 목적이 **제한된 인원 중 고위험 고객을 우선 선별하는 것**이므로 무가중치 LightGBM을 선택했다.

## 6. 최종 모델 비교

최종 후보:
- Logistic Regression
- Random Forest
- XGBoost
- LightGBM

Dummy는 학습 모델이 아니라 기준선으로만 사용했다.

Nested CV + OOF 기준:

| 모델 | OOF PR-AUC | Precision | Recall | F1 | Top10 Lift |
|---|---:|---:|---:|---:|---:|
| **LightGBM** | **0.3172** | 0.3257 | 0.3269 | **0.3263** | **3.2847** |
| XGBoost | 0.2717 | 0.2879 | 0.3216 | 0.3038 | 3.0381 |
| Random Forest | 0.2367 | 0.2020 | 0.4361 | 0.2761 | 2.5626 |
| Logistic Regression | 0.1788 | 0.1932 | 0.3930 | 0.2590 | 2.3600 |
| Dummy | 약 0.097 | - | - | - | 1.0 |

최종 Champion:
```text
LightGBM
```

선정 이유:
- 가장 높은 OOF PR-AUC
- 가장 높은 Top10 Lift
- 복잡한 비선형 관계와 Feature 상호작용을 효과적으로 학습

> Feature 실험의 A3 PR-AUC `0.317581`과 최종 Nested CV의 `0.3172`는 평가 프로토콜이 다르므로 완전히 같은 숫자로 비교하지 않는다.

## 7. Threshold 선택

Threshold는 Test에서 다시 찾지 않고 **Train OOF 예측에서 결정**했다.

```text
Nested CV
    ↓
OOF 예측
    ↓
F1 기준 Threshold 선택
    ↓
Threshold = 0.1305
    ↓
Test에 그대로 적용
```

이렇게 모델 선택과 Threshold 결정을 Test 평가와 분리했다.

## 8. 최종 Test 결과

LightGBM:

| 지표 | 결과 |
|---|---:|
| PR-AUC | **0.3516** |
| ROC-AUC | 0.7110 |
| Precision | 0.2672 |
| Recall | 0.4507 |
| F1 | 0.3355 |
| Top 10% Recall | **0.3521** |
| Top 10% Lift | **3.51배** |

Confusion Matrix:
- TN: 2,287
- FP: 351
- FN: 156
- TP: 128

### 해석

Threshold 기준으로는 실제 이탈 고객의 약 **45.1%**를 포착했다.

예측 위험도 상위 10% 고객만 관리하면:
- 전체 이탈자의 약 **35.2%**가 포함됨
- 무작위 선정 대비 이탈 고객 밀도가 약 **3.51배** 높음

따라서 이 모델은 “누가 반드시 이탈한다”를 확정하는 모델보다 **누구에게 먼저 연락할지 우선순위를 정하는 모델**로 활용하는 것이 적절하다.

## 10. 최종 운영 방향

```text
고객별 이탈 위험도 산출
        ↓
위험도 순으로 정렬
        ↓
캠페인 가능 인원에 맞춰 Top 5%, 10%, 20% 선택
        ↓
고위험 고객 우선 관리
        ↓
실제 캠페인 결과로 효과 검증
```

현재 모델이 답하는 질문:

> **누구에게 먼저 연락할 것인가?**

현재 모델만으로 답할 수 없는 질문:
- 연락하면 실제로 이탈을 막을 수 있는가?
- 어떤 혜택이 가장 효과적인가?
- 고객별 캠페인 비용 대비 수익은 얼마인가?

이 부분은 실제 캠페인 데이터, 고객 가치, A/B Test 또는 Uplift Modeling이 추가로 필요하다.
