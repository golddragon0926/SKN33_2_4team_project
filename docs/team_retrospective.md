# 팀원 회고

## 김정재

수업에서 배운 내용을 실제 데이터에 적용해 보면서, 이론으로만 알던 과정들을 더 이해할 수 있었습니다.

그리고 팀장으로 프로젝트를 진행하면서 팀원들과 의견을 맞추고 하나씩 완성해 가는 과정에서 많이 배웠고, 특히 예상대로 되지 않는 부분을 직접 수정하고 해결해 나가면서 프로젝트를 끝까지 완성하는 경험이 기억에 남았습니다.

## 김영석

### 🌟 Keep (잘했던 점 & 보람을 느낀 점)

#### 직관적이고 가독성 높은 B2B 대시보드 UI/UX 구현

단순히 분석 결과를 나열하는 것에 그치지 않고, `inject_common_css()`를 통한 전역 스타일링과 커스텀 메트릭 카드(`st.metric`)를 적용하여 대형 수치 중심의 시원한 시각적 가독성을 확보했습니다.

#### 사용자 경험(UX) 중심의 What-If 실시간 시뮬레이터 구축

복잡한 AI 모델의 추론 과정을 현장 담당자(AM)가 쉽게 이해하고 조작할 수 있도록 핵심 요인 조절 슬라이더와 실시간 위험도 변동 차트를 연동하여 B2B 현장 활용성을 크게 높였습니다.

#### 체계적인 Git & GitHub 협업 체계 구축 및 운영

`CONTRIBUTING.md` 작성 및 브랜치 전략(Feature Branching)을 수립하고, 팀원 간 코드 충돌(Conflict)을 최소화하며 안전하게 메인 코드베이스로 통합되도록 Git 리포지토리를 관리했습니다.

### ⚠️ Problem (아쉬웠던 점 & 고민했던 순간)

#### Streamlit 세션 상태 및 브라우저 캐싱 문제 대응

페이지 전환 시 전역 CSS가 풀리거나 CSS 변경 사항이 브라우저 캐시에 남아 즉각 반영되지 않는 현상이 발생하여, 각 서브 페이지마다 표준화된 CSS 주입 구조 및 안내 방식을 잡는 데 시행착오를 겪었습니다.

## 김혜진

이번 프로젝트는 모델링을 맡으면서 여러 번 다시 검토하고 의심하는 과정이 중요한 것을 느꼈다. 그리고 팀원들과 잘 맞물려 돌아간다는 걸 느꼈다. 급급하게 프로젝트를 진행하기보다 서로서로 도움을 주고 받는 협업의 좋은 예를 경험한 것 같다. 발표자료를 만들며 청자가 이해하기 쉽게 전달하는 것에서도 고민을 계속 하면서 프로젝트를 마무리하기 전 한 번 더 생각하고 학습한 계기가 된 것 같다. 모든 일이 그렇듯 조금의 아쉬움도 있지만 그만큼 다음엔 더 잘 해야지라는 다짐과, 더 잘 할 수 있겠다는 자신감이 생긴 프로젝트였다.

## 신가을

모델링을 진행하면서 실제 머리로 이해하고 있는 흐름과 많은 부분의 차이가 있음을 경험해 볼 수 있었다. 비즈니스 전략 도출을 위해 무엇을 중점으로 바라볼 지 각자의 시각에 따라 모델 선정 및 해석과 도출하는 과정이 다양할 수 있다는 것이 재미있는 경험이었던 것 같다. 팀원들 각자가 자기가 맡은 분야만이 아닌 프로젝트 전반에 각자의 고민과 아이디어를 제시하고 조정하는 모습 역시 인상 깊었다. 프로젝트 팀 안내와 준비 기간 중 연휴가 있어 다소 촉박하지 않나 라는 생각에 조급할 수 있었으나, 그럼에도 불구하고 팀원들과의 원활한 협력으로 잘 마무리 한 것 같다.
